# Contributing to Chitragupta

## Commit Message Convention

```
<Type>: [TASK-NNN —] <short description>
```

**Types:**

| Type | Changelog section | When to use |
|------|-------------------|-------------|
| `Feat` | Added | New feature or capability |
| `Fix` | Fixed | Bug fix |
| `Docs` | Documentation* | Documentation-only changes |
| `Refactor` | Changed | Code restructure without behavior change |
| `Security` | Security | Security-relevant fixes |
| `Remove` | Removed | Removal of feature or API |
| `Chore` | *(omitted)* | Tooling, CI, dependency updates |

\* `Documentation` is a project-specific extension to Keep a Changelog's standard sections.
`Chore`, `Test`, and `CI` commits are omitted from the user-facing changelog.

Capitalize the type. `TASK-NNN —` is optional for non-task commits (hotfixes, tooling).

**Examples:**
```
Feat: TASK-110 — Add CSV export for chargeback reports
Fix: TASK-109 — Correct billing period boundary calculation
Docs: Update configuration reference for new metrics backend
Chore: Bump mkdocs-material to 9.6
```

## Changelog

`CHANGELOG.md` is generated by [git-cliff](https://git-cliff.org/).
**Do not edit `CHANGELOG.md` manually** — content is derived entirely from commit messages.

## Release Process

1. Ensure all commits on `main` for this release use the type prefixes above.
2. Generate and commit the updated changelog (pass `--tag` so commits are assigned to the version, not `[Unreleased]`):
   ```bash
   uv sync --group docs
   uv run git-cliff --config cliff.toml --tag vX.Y.Z --output CHANGELOG.md
   git add CHANGELOG.md
   git commit -m "Docs: Update CHANGELOG.md for vX.Y.Z"
   git push origin main
   ```
3. Tag and push:
   ```bash
   git tag vX.Y.Z
   git push origin vX.Y.Z
   ```
4. The `docs.yml` workflow runs automatically:
   - Generates release notes for this tag (git-cliff `--latest`)
   - Creates a GitHub Release with the generated notes as body
   - Deploys the versioned MkDocs site via `mike`

Pre-release tags (e.g. `v1.0.0-rc.1`) are automatically marked as pre-release on GitHub.

## Local Changelog Preview

```bash
uv sync --group docs

# Preview what the next release notes would look like
uv run git-cliff --config cliff.toml --unreleased

# Regenerate full CHANGELOG.md for a specific version (use before tagging)
uv run git-cliff --config cliff.toml --tag vX.Y.Z --output CHANGELOG.md
```
