# Chargeback Grafana Dashboards

Grafana dashboards for visualizing chargeback data from the SQLite database.

## Prerequisites

- Docker and Docker Compose installed
- Chargeback database file (SQLite) generated by running the pipeline (`chargeback.db`)

## Quick Start

```bash
# Set database path (optional — defaults to ./data/chargeback.db)
export CHARGEBACK_DB_PATH=/path/to/your/chargeback.db

# Start Grafana
docker compose up -d

# Access dashboards
open http://localhost:3000
# Login: admin / password
```

Grafana will auto-install the `frser-sqlite-datasource` plugin and provision both dashboards on first start. Allow 30–60 seconds for startup.

## Configuration

| Variable | Default | Description |
|---|---|---|
| `CHARGEBACK_DB_PATH` | `./data/chargeback.db` | Host path to the SQLite database |
| `GF_SECURITY_ADMIN_PASSWORD` | `password` | Grafana admin password |

To override the admin password:

```bash
GF_SECURITY_ADMIN_PASSWORD=mysecretpassword docker compose up -d
```

The port mapping is fixed to `3000:3000`. To change it, edit `docker-compose.yml`.

## Dashboards

### Chargeback Overview

High-level cost visualization. Use this for monitoring and reporting.

Panels:
- **Data Availability** — time series showing which dates have chargeback records
- **Total Cost / Usage Cost / Shared Cost** — aggregate stats for the selected period
- **Cost by Environment** — pie chart via resource parent_id join
- **Cost by Product Category** — pie chart
- **Cost by Product Type** — pie chart
- **Cost by Identity (Top 10)** — pie chart with display name resolution
- **Cost by Resource (Top 10)** — pie chart with display name resolution
- **Cost Over Time** — stacked bar chart by product category

### Chargeback Details

Drill-down tables for investigation and auditing.

Panels:
- **Chargeback Details** — full fact/dimension join with display names (1000 row limit)
- **Billing Data** — raw billing table aggregated by resource and product
- **Unallocated Costs** — rows where `identity_id = 'UNALLOCATED'`
- **Allocation Issues** — rows where `allocation_detail` is non-empty (failed/partial allocations)
- **Resources** — resource inventory with type, environment (parent_id), and status
- **Identities** — identity inventory with type

## Template Variables

Both dashboards share the same cascading filters:

1. **ecosystem** — single-select, required; selects which ecosystem's data to show
2. **tenant_id** — multi-select; cascades from ecosystem
3. **environment** — multi-select; cascades from tenant (via `resources.parent_id`)
4. **product_category** — multi-select; cascades from tenant
5. **product_type** — multi-select; cascades from product category

Select "All" for multi-select variables to include all values.

## Customization

### Modifying dashboard panels

Edit the JSON files in `grafana/provisioning/dashboards/`. Each panel is a JSON object in the `panels` array. After editing, restart Grafana or use "Reload provisioned dashboards" in Grafana UI (Configuration → Provisioning).

### Adding new panels

1. Open the dashboard in Grafana UI and add a panel interactively
2. Export the dashboard JSON (Share → Export → Save to file)
3. Replace the corresponding JSON file in `grafana/provisioning/dashboards/`

### Changing the datasource

The datasource UID `chargeback-sqlite` is referenced in all panel queries. If you change it in `datasource.yml`, update all panel `datasource.uid` fields in both dashboard JSONs.

See [Grafana documentation](https://grafana.com/docs/grafana/latest/dashboards/) for general dashboard editing guidance.

## Troubleshooting

**"No data" in all panels**
- Verify the database path: `ls -la ${CHARGEBACK_DB_PATH:-./data/chargeback.db}`
- Check the pipeline has run and produced output: `sqlite3 chargeback.db "SELECT COUNT(*) FROM chargeback_facts;"`
- Confirm the time range in Grafana covers the dates in your data

**Datasource connection error**
- Verify the SQLite plugin installed: check Grafana logs with `docker compose logs grafana | grep plugin`
- Check file permissions: the database file must be readable by the Grafana container (`chmod 644 chargeback.db`)
- Confirm the volume mount path matches: `docker compose exec grafana ls /var/lib/grafana/data/`

**Template variable dropdowns are empty**
- The database may be empty — run the pipeline first
- The ecosystem variable has no data — check `SELECT DISTINCT ecosystem FROM chargeback_dimensions;`
- Filters may be too restrictive — reset variables to their defaults

**Grafana fails to start**
- Check logs: `docker compose logs grafana`
- The SQLite plugin install requires internet access on first run; ensure Docker has outbound connectivity
- Port 3000 may be in use: `lsof -i :3000`
